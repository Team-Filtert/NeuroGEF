GAME STATE

All runtime state will be stored on the GameState autoload.

```
# Rough shape
extends Node

var world: WorldState # Snapshot data container for more meta-like data, like player position or current scene path.
var party: PartyState # Live data container for the current party.
var inventory: InventoryState # Live data container for inventory, this along with items and stuff warrant their own design doc.
```

Keep in mind this supposed to have serialization/deserialization and other helper methods.

THE PLAYER

I've decided that the player is separate from levels on the tree, being on it's own PlayerRoot node separate from LevelRoot, and is repositioned when needed, so it always stays on the tree while on gameplay.

The class itself is dumb and just helps with visuals.

If something external ever needs a reference to the player, that's coordinated through methods (E.G.: PlayerManager.get_player(), PlayerManager.position_player() or signals), NEVER directly.
